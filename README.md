# 🥗NutriScan — AI Food Recognition & Nutrition Tracker

NutriScan is a deep learning application that identifies food from photos and tracks daily nutrition. It supports single dish recognition, multi-dish thali detection, and manual meal logging — all through a Streamlit web interface.

---

## Features

- **AI Food Recognition** — identifies 182 food categories including Indian and international cuisines
- **Thali Detection** — detects multiple dishes in a single plate photo using a grid-crop strategy
- **Nutrition Tracking** — shows calories, carbs, protein, fat, fiber, sugar, and sodium per meal
- **Gram-based Scaling** — nutrition values scale proportionally to the actual quantity you ate
- **Manual Entry** — log any dish manually even without a photo; enter custom nutrition for unlisted foods
- **Daily Goals** — customisable daily nutrition targets with colour-coded progress bars
- **Per-dish Editing** — correct wrong predictions, adjust quantities, add or remove dishes

## Dataset

The model is trained on a merged dataset:

- **Food-101** — 101 standard food categories, ~1,000 images per class
- **Indian Food** — 81 additional Indian dish categories, merged into the Food-101 `images/` folder

Total: **182 classes**, ~105,000 images

The Indian food images were sourced separately and placed as subfolders inside the Food-101 `images/` directory alongside the original classes. The `meta/classes.txt` file was updated to include all 182 class names.

---

## Model Architecture

**Backbone:** EfficientNet-B3 (pretrained on ImageNet)

The final classification head is replaced with:
```
Dropout(0.5) → Linear(1536 → 182)
```

Input resolution: 300×300 pixels

Alternative backbones supported: ResNet-50, EfficientNet-B0, ViT-B/16

---

## Training Strategy

### Two-phase fine-tuning
- **Epochs 0–2 (warmup):** Only the classification head is trained (backbone frozen). LR = 1e-3
- **Epoch 3 onwards:** All layers unfrozen. LR = 3e-5 with cosine annealing to 1e-6

### Class imbalance handling
Indian food classes have significantly fewer images than Food-101 classes. Two mechanisms address this:

1. **Class-aware augmentation** — classes with fewer than 150 images get a stronger augmentation pipeline (TrivialAugmentWide, wider crop range, more colour jitter, vertical flipping, heavier random erasing)
2. **Weighted sampling** — minority classes are oversampled by a factor of 3× using `WeightedRandomSampler`, so the model sees them proportionally during every epoch

### Augmentation
Majority classes (≥150 images):
- RandomResizedCrop, RandomHorizontalFlip, RandAugment, RandomRotation(15°), ColorJitter, RandomErasing

Minority classes (<150 images):
- RandomResizedCrop (wider scale), RandomHorizontalFlip + RandomVerticalFlip, TrivialAugmentWide, RandomRotation(30°), stronger ColorJitter, RandomGrayscale, heavier RandomErasing

### MixUp + CutMix
After the warmup phase, each training batch has a 50% chance of applying either MixUp (α=0.4) or CutMix (α=1.0). This improves generalisation by training the model on blended images with soft labels.

### Loss
CrossEntropyLoss with label smoothing (ε=0.1) — prevents overconfidence on training data.

### Optimiser
AdamW with weight decay 0.01. Gradient clipping at max_norm=1.0.

## Training

1. Download Food-101 from [https://data.vision.ee.ethz.ch/cvl/datasets_extra/food-101/](https://data.vision.ee.ethz.ch/cvl/datasets_extra/food-101/)
2. Place Indian food images as subfolders inside `food-101/images/` and update `meta/classes.txt`
3. Edit the config section at the top of `food101_train.py`:

```python
MODEL       = "efficientnet_b3"
FOOD101_DIR = r"path/to/food-101"
INDIAN_DIR  = ""        # leave blank if already merged into FOOD101_DIR
EPOCHS      = 30
BATCH_SIZE  = 32
SAVE_DIR    = r"path/to/save/results"
UTILS_DIR   = r"path/to/utils"
```

4. Run training:
```bash
python food101_train.py
```

Training outputs saved to `SAVE_DIR/efficientnet_b3/`:
- `best_model.pth` — checkpoint with best validation accuracy
- `final_weights.pth` — weights at the end of training
- `training_curves.png` — loss and accuracy plots
- `history.json` — epoch-by-epoch metrics

After training, `best_model.pth` is automatically copied to `UTILS_DIR/` for the app to load.

## App Usage

### Single dish mode
Upload a photo of one dish. The model returns top-5 predictions with confidence scores. Select the correct dish if the top prediction is wrong, enter the quantity in grams, and log it to your daily tracker.

### Thali / multi-dish mode
Upload a photo of a full thali or any plate with multiple items. The app splits the image into 2×2 and 3×3 grids, runs inference on each crop, and aggregates results into a dish list with estimated plate-area percentages. Each dish can be corrected, quantity adjusted, or removed. New dishes can be added manually.

### Manual entry mode
No photo needed. Select dishes from the full 182-class list and enter gram quantities. For anything not in the list, choose **Other** and enter the nutrition values manually.

### Daily goals
Click **Customise Daily Goals** to set personal calorie and macronutrient targets. Progress bars turn amber at 75% and red at 100%.

## Thali Detection — How It Works

The model is a single-label classifier (trained to predict one dish per image). Multi-dish detection is achieved at inference time without retraining:

1. The uploaded image is divided into a 2×2 grid (4 crops) and a 3×3 grid (9 crops)
2. Each crop is classified independently
3. Results are aggregated by class: confidence-weighted area fractions are accumulated per dish
4. Classes below the confidence threshold (default 0.20) are discarded
5. Remaining area fractions are normalised to 100% and used to estimate portion sizes

This approach works because different katoris in a thali occupy distinct spatial regions of the image.

## Possible Improvements

The code is fully functional but these are the next steps that would improve it further:

**Model accuracy**
- Train for more epochs (50–60) — accuracy typically still improves after epoch 30 on this dataset size
- Use EfficientNet-B4 or B5 for higher capacity (at the cost of more VRAM and training time)
- Add test-time augmentation (TTA): average predictions across multiple crops/flips of the same image at inference time — typically adds 1–2% accuracy without retraining
- Try a learning rate warmup scheduler in the first phase instead of a fixed LR

**Data**
- Collect more images for the 81 minority classes (the single biggest accuracy lever)
- Use web scraping or dataset APIs to expand Indian food classes to 500+ images each

**Thali detection**
- The current grid approach is a heuristic. For a real product, training a proper object detector (YOLOv8 or DETR) on a thali dataset with bounding box annotations would give much more accurate multi-dish segmentation and portion estimation

**App**
- Add a history page that persists the daily log across browser sessions (currently resets on page reload) using a local SQLite database or a JSON file
- Export nutrition log as CSV or PDF
- Add a BMR/TDEE calculator to personalise the default calorie goal
- Add meal photos thumbnails in the daily log

## Notes

- Nutrition values in `nutrition_values.json` are approximate common-serving estimates. They vary by recipe, restaurant, and brand and are not suitable for medical nutrition planning.
- The model was trained on web-scraped food images. Real-world photos (especially home-cooked food or unusual presentations) may produce lower confidence scores — use the correction dropdown in those cases.
- On Windows, if `streamlit run` is blocked by Application Control policy, use `python -m streamlit run food101_app.py` instead.
